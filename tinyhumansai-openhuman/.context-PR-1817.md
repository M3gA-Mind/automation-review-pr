## PR Description
Fix five panic-prone `&command[..command.len().min(80)]` log-truncation slices in `SecurityPolicy::validate_command_execution` that aborted the core thread when the command contained a multi-byte UTF-8 char straddling byte 80. Routes each site through the existing `crate::openhuman::util::floor_char_boundary` helper — the same helper that fixed the equivalent body-preview bug in #1751. Adds two regression tests in `policy_tests.rs` covering the real Sentry repro (CJK in a `cmd /c` command) and a short multi-byte edge case.

**Problem**: [OPENHUMAN-TAURI-GW](https://tinyhumans.sentry.io/issues/OPENHUMAN-TAURI-GW) fires a `level=fatal`, `mechanism=panic`, `handled=no` event on Windows whenever a Chinese / Japanese / Korean user's command happens to put a multi-byte UTF-8 character across byte 80 of the command string.

**Solution**: Replace each `&command[..command.len().min(80)]` with `&command[..floor_char_boundary(command, 80)]`.
