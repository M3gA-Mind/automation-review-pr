## PR Description
Makes the #2010 per-model embedding sidecar tables **live**: ingestion and retrieval now read/write `mem_tree_{chunk,summary}_embeddings` scoped by an active model signature, instead of the legacy single `embedding` columns.

Key components:
1. **Single-source signature formatter** (`format_embedding_signature`) + probe-stable, config-derived `active_embedding_signature` (keyed off intended selection, not Ollama-health probe)
2. **Cutover**: chunk + summary read AND write paths delegate to the sidecar at the active signature, in-transaction with existing writes
3. **One-shot migration** (`PRAGMA user_version`-gated): copies dim-matching legacy vectors into sidecar (legacy columns kept for vN+1 drop)
4. **ReembedBackfill job**: re-embeds rows missing a vector at the active signature, bounded via `scheduler_gate` single-LLM-slot, self-continues via `Defer`
5. **Switch trigger**: embedder change (memory-settings AND model-settings/AIPanel) triggers the backfill
6. **Frontend**: `memory_backfill_status` RPC + advisory AIPanel modal surfaces progress
7. **`backfill_in_progress` flag**: #1365 first-person/subconscious layer treats empty recall during backfill as "not searched yet"
8. **Composio test deflake**: factory-routing tests converted from async+env-var to sync pure-function calls
9. **CEF e2e test deflake**: serializes env-mutating tests via `Mutex`
