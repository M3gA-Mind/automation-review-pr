+# Polymarket Integration (Read + Trading)
+
+This document describes the Polymarket integration for issue #1398.
+
+## Scope
+
+The `polymarket` tool now supports both market browsing and trading workflows over:
+
+- Gamma API (`https://gamma-api.polymarket.com`)
+- CLOB API (`https://clob.polymarket.com`)
+
+Supported read actions:
+
+- `list_markets`
+- `get_market`
+- `list_events`
+- `get_orderbook`
+- `get_price`
+- `get_positions`
+- `get_balance`
+- `get_open_orders`
+- `get_usdc_allowance`
+
+Supported write actions:
+
+- `place_order`
+- `cancel_order`
+
+## Architecture
+
+Implementation lives in `src/openhuman/tools/impl/network/polymarket.rs` with helper modules:
+
+- `clob_auth.rs`: L1 credential derivation + L2 HMAC headers
+- `polymarket_orders.rs`: EIP-712 order typed-data signing
+
+Key runtime behavior:
+
+- Layer-2 API credentials are derived on first authenticated call and cached.
+- Derived credentials are persisted to `integrations.polymarket.derived_clob_credentials` (plain config fallback until secret-store migration lands).
+- Order placement fetches `GET /nonce?user=<eoa>` before signing to avoid replay/nonce mismatch.
+
+## Authentication
+
+### L1 handshake
+
+On first authenticated call, the tool signs an EIP-712 `ClobAuth` typed-data message using the wallet's EVM key.
+
+1. Build `ClobAuth { address, timestamp, nonce: 0, message }` with `domain.name = "ClobAuthDomain"`, `domain.chainId = 137`, `domain.verifyingContract = <clob_exchange_contract>`.
+2. POST `{clob_base_url}/auth/api-key` with the address, timestamp, nonce, and hex signature.
+3. If the endpoint returns 404 or an error, fall back to `GET {clob_base_url}/auth/derive-api-key?...` with the same query parameters.
+4. Parse response `{ apiKey, secret, passphrase }`.
+
+### L2 HMAC signing
+
+Every authenticated CLOB request carries three extra headers:
+
+- `POLY-API-KEY`: from derived credentials
+- `POLY-PASSPHRASE`: from derived credentials
+- `POLY-TIMESTAMP`: current unix seconds
+- `POLY-SIGNATURE`: `HMAC-SHA256(secret, timestamp + method + requestPath + body)` base64-encoded (URL-safe)
+
+### EIP-712 order signing
+
+`place_order` builds a `CTFExchangeOrder` EIP-712 typed-data struct and signs it with the wallet key. Domain: `PolymarketCTFExchange`, chain 137, contract `<clob_exchange_contract>`.
+
+## Headers and nonce rules
+
+- Nonce for L1 handshake: always `0`.
+- Nonce for order placement: fetched per-order via `GET /nonce?user=<address>` from CLOB API.
+- Order nonce is string-typed in the API response; the tool handles both `"<number>"` and bare `<number>` JSON values.
+
+## Error and retry behavior
+
+- **4xx**: Not retried. Returns error with status + truncated body (max 240 chars, multibyte-safe).
+- **429 / 5xx**: Retried up to 3 times with 500 ms fixed backoff.
+- **Timeout**: `timeout_secs` (default 15) applies per-request; connect timeout fixed at 10 s. Deadline exceeded returns explicit error.
+
+## Permissions
+
+- Read actions: no gating beyond `integrations.polymarket.enabled`.
+- Write actions (`place_order`, `cancel_order`): require `arguments.approved == true`. If missing, the tool returns an approval-required message and issues **no HTTP**. This is a temporary stopgap until the shared approval gate (#1339) is available.
+
+## Configuration
+
+All fields live under `[integrations.polymarket]` in `config.toml`:
+
+| Field | Default | Purpose |
+|---|---|---|
+| `enabled` | `false` | Master toggle |
+| `gamma_base_url` | `https://gamma-api.polymarket.com` | Gamma REST endpoint |
+| `clob_base_url` | `https://clob.polymarket.com` | CLOB REST endpoint |
+| `timeout_secs` | `15` | Per-request deadline |
+| `eoa_address` | (none) | Override default wallet address |
+| `polygon_rpc_url` | `https://polygon-rpc.com` | Polygon JSON-RPC for `eth_call` |
+| `usdc_contract` | `0x2791Bca1f2de4661ED88A30C99A7a9449Aa84174` | USDC.e bridged on Polygon |
+| `clob_exchange_contract` | `0x4bFb41d5B3570DeFd03C39a9A4D8dE6Bd8B8982E` | Polymarket CLOB exchange |
+| `derived_clob_credentials` | (none) | Cached L2 API credentials (auto-populated) |
