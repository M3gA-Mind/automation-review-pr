## PR Description
Four `tree_jobs_worker` tasks were racing into `memory::tree::store::with_connection` on cold start, triggering ~19k Sentry events/day across three SQLite WAL/SHM bootstrap codes. Fix: gate `apply_schema` behind a process-wide `Mutex<HashSet<PathBuf>>` so only the first caller per DB path runs WAL setup + SCHEMA + ALTER TABLE migrations; subsequent callers skip. Retry the open+init flow up to 3x on three transient cold-start codes (14 / 1546 / 4874) with 50→150→500ms backoff. Add regression test spawning 8 threads against cold tempdir DB.

Sentry issues addressed:
- OPENHUMAN-TAURI-HH: 14168 events — IOERR_TRUNCATE (code 1546)
- OPENHUMAN-TAURI-ZM: 3013 events — IOERR_SHMMAP (code 4874)
- OPENHUMAN-TAURI-MB: 2177 events — CANTOPEN (code 14)

+    fn is_transient_sqlite(e: &(dyn std::error::Error + 'static)) -> bool {
+        if let Some(rusqlite::Error::SqliteFailure(ffi, _)) = e.downcast_ref::<rusqlite::Error>() {
+            return matches!(
+                ffi.extended_code,
+                SQLITE_CANTOPEN | SQLITE_IOERR_TRUNCATE | SQLITE_IOERR_SHMMAP
+            );
+        }
+        false
+    }
+    if is_transient_sqlite(err.root_cause()) {
+        return true;
+    }
+    let mut src: Option<&(dyn std::error::Error + 'static)> = Some(err.as_ref());
+    while let Some(cur) = src {
+        if is_transient_sqlite(cur) {
+            return true;
+        }
+        src = cur.source();
+    }
+    false
+}
