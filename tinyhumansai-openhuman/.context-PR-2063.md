## PR Description
Adds `ExpectedErrorKind::LoopbackUnavailable` variant to classify loopback `Connection refused` errors from the in-process core boot-window race. Sibling components (frontend RPC relay, integrations/composio HTTP clients) reach `127.0.0.1:<port>` before the embedded core's listener finishes binding, producing TCP `Connection refused` that currently flows through `report_error_message` → Sentry. This PR demotes them to `debug!` breadcrumbs.

Targets OPENHUMAN-TAURI-R5 (~2.5k events, `integrations.get`) and OPENHUMAN-TAURI-R6 (~2.5k events, `rpc.invoke_method` re-wrap). Both share `trace_id=6ebf5b62748d5144e541e2cddeabbbd0`.

Three commits:
1. Enum variant + `report_expected_message` arm
2. `is_loopback_unavailable` matcher + ladder precedence (before `NetworkUnreachable`)
3. Tests — verbatim R5/R6 bodies, cross-platform errno, precedence guard, negative cases
