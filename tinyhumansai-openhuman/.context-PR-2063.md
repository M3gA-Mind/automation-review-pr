## PR Description
Adds `ExpectedErrorKind::LoopbackUnavailable` variant to classify in-process-core boot-window race conditions where sibling components (frontend RPC relay, integrations/composio HTTP clients) hit `127.0.0.1:<port>` before the embedded core's listener finishes binding and get TCP `Connection refused`.

Three commits:
1. New enum variant + match arm in `report_expected_message` (uses `tracing::debug!` with metadata-only structured fields)
2. `is_loopback_unavailable` conjunctive matcher + ladder precedence insertion before `is_network_unreachable_message`
3. Tests — verbatim R5/R6 bodies, cross-platform errno variants, precedence guard, negative cases, smoke test through `report_error_or_expected`

Targets OPENHUMAN-TAURI-R5 (~2.5k events) and OPENHUMAN-TAURI-R6 (~2.5k events) — both the same loopback connect-refused shape at different emit sites.
