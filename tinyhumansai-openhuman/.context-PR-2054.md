## PR Description
Adds `[Cross-chat context]` block to the agent prompt drawn from the same user's other JSONL chat threads, with provenance tags.

Two complementary retrieval paths:
1. **Primary (live chats):** JSONL substring scan over user's `workspace/memory/conversations/threads/*.jsonl`, scoring by query-term overlap. Filtered by `min_relevance_score`, capped at `CROSS_CHAT_LIMIT`, snippet-truncated, rendered with `chat:<hash>` provenance tags.
2. **Fallback (tests/headless):** When `workspace_dir` not wired, falls back to `Memory::recall(cross_session=true)` → existing FTS5 `episodic_cross_session_search`.

Key components:
- `RecallOpts.cross_session` field on the memory trait
- `ConversationStore::search_cross_thread_messages` — JSONL scan
- `episodic_cross_session_search` — FTS5 SQL helper
- `UnifiedMemory` cross-session recall integration
- `DefaultMemoryLoader.with_workspace_dir()` — wires JSONL path
- `build_context()` in memory_context.rs — harness-side integration
- `provenance_tag()` in util.rs — hashed session id rendering
- Session builder wires `config.workspace_dir` into the loader

+    /// Workspace scope is enforced by the store's `workspace_dir` — one

+        &self,

+        let _guard = CONVERSATION_STORE_LOCK.lock();

+            .split_whitespace()
+            .filter(|t| t.len() >= 3)
