**Findings**:
- [minor] `pii.rs:216` — PAN-IN regex `[A-Z]{5}\d{4}[A-Z]` (case-insensitive, no checksum, no keyword gating) is the only pattern without a secondary validation gate. Product codes, model numbers, or function names matching this format will be silently redacted. Suggest keyword gating (like Aadhaar) or explicit FP documentation.
- [minor] `pii.rs:247` — SCREEN DNI entry `\b\d{8}[A-Z]\b` is case-sensitive (no `(?i)`) while `DNI_RE` uses `(?i)`. A lowercase check letter (e.g., `12345678z`) fails both SCREEN checks and is never passed to `DNI_RE`. Fix: add `(?i)` to the SCREEN entry. All other letter-containing SCREEN entries already use `(?i)`.
